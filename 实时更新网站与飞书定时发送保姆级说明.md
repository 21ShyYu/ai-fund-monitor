# 实时更新网站与飞书定时发送保姆级说明

本文按你当前项目结构给出可直接执行的步骤，目标是：
1. 新闻时政热点正常抓取（标题/时间/摘要）
2. 网页 `www.iyushy.site` 持续更新
3. 飞书按定时任务自动发送
4. LLM 研判只使用“最近净值 + 当前时政新闻”，不参考 XGBoost 结果

## 1. 你这次修复后的行为

- 新闻抓取改为时政双源：
  - 中国政府网（要闻）
  - 人民网（时政）
- 新闻入库字段仍是：`title`、`summary`、`published_at`、`source`、`category`。
- LLM 输入已改为：
  - 每只基金最近净值序列 + 最新涨跌
  - 当前时政新闻列表
  - 明确禁止引用 XGBoost 结果
- XGBoost 结果仍用于：
  - 网页信号展示（`latest_signals`）
  - 飞书“XGBoost信号结果”

## 2. 先在服务器做一次完整自检（必须）

以下命令在服务器项目根目录执行（例如 `/root/ai-fund-monitor`）：

```bash
cd /root/ai-fund-monitor/worker
source .venv/bin/activate
python -m scripts.run_once
```

执行后检查 4 件事：

1. 飞书是否收到消息。
2. 日志是否出现 `news=` 且大于 0。
3. `data_exports/dashboard.json` 的 `news` 数组是否非空。
4. 网站首页新闻表格是否更新（最多等待前端缓存刷新窗口）。

如果你要立即看最新网页，不等缓存：
- 你的 Next.js 当前配置是 `revalidate = 1800`（约 30 分钟）
- 可在 Vercel 控制台手动 Redeploy 一次，立即刷新。

## 3. 确保网站能“自动更新”而不是只本地更新

关键是 worker 每次跑完要把 `data_exports/dashboard.json` 推到 GitHub 主分支。

检查 `worker/.env`：

```env
GIT_AUTO_PUSH=true
GITHUB_BRANCH=main
EXPORT_DIR=./data_exports
```

并确认服务器 git 具备 push 权限（SSH key 或 PAT 已配置）。

每次 `python -m scripts.run_once` 成功后会：
1. 写入最新 `data_exports/dashboard.json`
2. 自动 `git add/commit/push`
3. 网站通过 `DATA_BASE_URL` 拉取最新 JSON

## 4. 定时给飞书发送（也是定时更新网站）

编辑 crontab：

```bash
crontab -e
```

推荐定时（北京时间）：

```cron
SHELL=/bin/bash
PATH=/usr/local/sbin:/usr/local/bin:/usr/sbin:/usr/bin:/sbin:/bin
CRON_TZ=Asia/Shanghai

# 早盘前检查
30 8 * * * cd /root/ai-fund-monitor/worker && . .venv/bin/activate && python -m scripts.run_once >> /root/ai-fund-monitor/worker/runtime/cron.log 2>&1

# 午间
0 12 * * * cd /root/ai-fund-monitor/worker && . .venv/bin/activate && python -m scripts.run_once >> /root/ai-fund-monitor/worker/runtime/cron.log 2>&1

# 收盘后
10 15 * * * cd /root/ai-fund-monitor/worker && . .venv/bin/activate && python -m scripts.run_once >> /root/ai-fund-monitor/worker/runtime/cron.log 2>&1

# 晚间复盘
0 22 * * * cd /root/ai-fund-monitor/worker && . .venv/bin/activate && python -m scripts.run_once >> /root/ai-fund-monitor/worker/runtime/cron.log 2>&1
```

保存后验证：

```bash
crontab -l
tail -n 100 /root/ai-fund-monitor/worker/runtime/cron.log
```

## 5. 必须确认的环境变量

`worker/.env` 至少要有：

```env
DB_PATH=./worker/runtime/fund.db
MODEL_DIR=./worker/runtime/models
EXPORT_DIR=./data_exports
CONFIG_DIR=./shared/config

FEISHU_WEBHOOK=https://open.feishu.cn/open-apis/bot/v2/hook/xxxx

LLM_API_KEY=你的大模型Key
LLM_BASE_URL=你的模型兼容地址
LLM_MODEL=你的模型名
LLM_TIMEOUT_SEC=40

GIT_AUTO_PUSH=true
GITHUB_BRANCH=main
TZ=Asia/Shanghai
```

如果 `LLM_API_KEY` 为空，会出现：`LLM report failed: Missing LLM_API_KEY`。

## 6. 出现“有飞书但网页不更新”时怎么查

按顺序查：

1. `python -m scripts.run_once` 是否执行成功。
2. `data_exports/dashboard.json` 时间戳是否变化。
3. `git log -1` 是否出现自动提交。
4. `git push` 是否成功（看 cron.log）。
5. 前端 `DATA_BASE_URL` 是否指向正确仓库分支。
6. Vercel 是否成功部署，且无拉取 JSON 报错。

## 7. 你现在的关键结论

- 新闻抓取已恢复为实时时政热点抓取，不再依赖你之前清空的新闻 JSON。
- LLM 研判逻辑已与 XGBoost 解耦（仅看净值+时政新闻）。
- XGBoost 结果仍保留在网页和飞书展示链路中。
